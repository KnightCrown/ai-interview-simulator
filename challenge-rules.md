# Creator Challenge Rules (Agent Guide)

This document summarizes the key constraints and evaluation criteria for the OpenAI x Handshake Codex Creator Challenge. All project changes, features, and decisions should optimize for these rules.

---

## 1. Eligibility Constraints (High Level)

- Must be an individual entrant (US-based eligibility required)
- Must be 18+ years old
- Must be a college undergraduate or alumnus in a US university
- Must submit via Handshake AI showcase page
- Must submit between:
  March 24, 2026 – April 30, 2026 (11:59 PM PT)

---

## 2. Evaluation Criteria (VERY IMPORTANT)

All entries are scored using a weighted rubric.

### 2.1 Clarity (15%)
Focus: How understandable the project is

Target:
- Clear problem statement
- Clear user target
- Clear solution explanation

Avoid:
- Vague purpose
- Unclear users
- Confusing explanation

---

### 2.2 Usefulness / Value (25%)  **(HIGH WEIGHT)**
Focus: Real-world impact

Target:
- Solves a real problem
- Improves workflow or daily life
- Clear user benefit

Avoid:
- Novel but useless ideas
- No clear application

---

### 2.3 Creativity (20%)
Focus: Originality

Target:
- Fresh concept or twist
- Interesting application of tech
- Non-obvious use case

Avoid:
- Generic clones
- Slight variations of common apps

---

### 2.4 Execution (25%)  **(HIGH WEIGHT)**
Focus: Functionality and reliability

Target:
- Fully working product
- Stable core features
- No major bugs

Avoid:
- Broken flows
- Partial implementations
- Demo-only concepts

---

### 2.5 Polish & Thoughtfulness (15%)
Focus: UX and product feel

Target:
- Clean UI/UX
- Thoughtful edge cases
- Intuitive user flow
- Feels like a real product

Avoid:
- Rough or unfinished UI
- Confusing interactions

---

## 3. Finalist Pitch Video (Top 10 Only)

If selected:

You must submit a 3-minute pitch video.

Scoring bonus (added to Round 2 score):

### Pitch Quality Score (1–5)

- 1: Unclear, confusing, no demo clarity
- 2: Basic explanation, incomplete story
- 3: Solid explanation + working demo
- 4: Strong, engaging, clear value + demo
- 5: Exceptional, polished, compelling, memorable

---

## 4. What Wins the Contest (Practical Guidance)

Winning projects usually have:

- A clearly defined user and pain point
- A “this could be a real startup/product” feel
- End-to-end working functionality
- Strong UX polish
- A simple but powerful idea executed extremely well

---

## 5. Disqualification Risks

Avoid:
- Broken or incomplete app
- Missing submission fields
- Automated spam entries
- Misleading or unclear descriptions
- Not meeting eligibility requirements

---

## 6. Strategy Notes for Developers

To maximize score:

### Prioritize:
- Working end-to-end flow over feature count
- UX clarity over complexity
- Real-world usefulness over novelty alone
- Stability over experimental features

### Optimize description writing:
Even if the app is strong, unclear descriptions reduce score significantly.

---

## 7. Core Principle

> A simple, fully working, useful product beats a complex unfinished idea.
